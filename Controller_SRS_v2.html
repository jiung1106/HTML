SECTION 01

개요 (Overview)

본 소프트웨어는 Panasonic MSMF5AZl1S1 AC 서보 드라이버를 ROS2 환경에서 제어하기 위한 MinasA6Modbus 클래스이다. Modbus RTU 프로토콜에 따라 드라이버의 레지스터 및 코일에 대한 요청 패킷을 생성하고 응답을 파싱하는 기능을 제공하며, 직렬 통신 자체와 ROS2 노드 연동은 클래스 외부(Transport 레이어, ROS2 브리지 노드)의 책임으로 둔다.

목적

MinasA6Modbus 클래스를 이용해 ROS2에서 모니터링 및 제어가 가능한 API를 제공한다.

3.2 CRC-16 알고리즘

RS-485 통신 프로토콜 규격에 맞는 CRC-16 방식을 사용해 프레임 마지막 2바이트에 Little-Endian으로 부착된다.

def CRC16(data: bytes) -> int:
    crc = 0xFFFF
    for byte in data:
        crc ^= byte
        for _ in range(8):
            if crc & 0x0001:
                crc = (crc >> 1) ^ 0xA001
            else:
                crc >>= 1
    return crc

4.4 블록 데이터 레지스터 (FC 10, 0x4800~)

블록 번호 1개당 4워드를 차지한다. 앞 2워드는 32bit 블록 커맨드(동작방식+속도/가속/감속 인덱스+방향+천이조건), 뒤 2워드는 32bit 블록 데이터(인수7, 절대/상대 이동량 등)다. block_no=0의 오프셋 규칙만 PANATERM readback으로 직접 검증됐다.

set_target_pos()는 절대 위치 제어(cmd_code=0x02) 블록을 위한 전용 래퍼로, 인수로 받은 목표 위치(원점 기준)에 __HOME 오프셋을 더한 값을 arg7(절대 위치)로 기록한다. 즉 arg7 = position + HOME.

· 주의:
- block_no가 0이 아닌 경우의 4800h+block_no×4 오프셋 공식은 아직 PANATERM으로 대조 검증된 적이 없어 다른 블록을 쓰기 전 get_block_cmd(block_no)로 한 번 더 확인할 필요가 있음.
- 현재 절대 위치 제어 시 Err94.2(원점복귀 없이 절대위치 제어 시도)가 발생하는 이슈가 진행 중이며, 근본 해결책으로 모터(MSMF5AZL1S1, 23bit 절대 엔코더)를 절대 엔코더 모드(Pr0.15=0)로 전환하는 방안을 검토 중이다.

3-1코일 0x0060에 ON/OFF 값을 써서 서보를 켜고 끈다.CMD_SV_ON() / CMD_SV_OFF()
3-2블록 동작 전에 속도/가속/감속 테이블을 먼저 채워야 한다.set_speed() / set_acc() / set_dec()
3-3cmd_code와 인수(arg1~arg7)만 바꾸어 JOG/위치결정/원점복귀 등 모든 블록 동작을 동일한 함수로 표현한다.set_block(block_num, cmd_code, arg1~arg7)
3-4절대 위치 제어(cmd_code=0x02) 블록은 목표 위치에 HOME 오프셋을 자동으로 더해 생성하는 전용 함수로 표현한다.set_target_pos(position, block_num, speed_idx, accel_idx, decel_idx, transition)

3-7감속정지 코일(0x0124)을 켠 뒤 끄고, STROBE를 OFF로 정리한다. ON/OFF가 별도 함수로 분리되어 있다.CMD_DECEL_STOP() → CMD_DECEL_STOP_OFF() → CMD_STROBE_OFF()
3-8get_busy()가 True인 동안 set_block()/set_target_pos()를 다시 호출하는 것은 정의되지 않은 동작이다. 호출자는 BUSY가 False임을 확인한 뒤 다음 블록을 설정해야 한다.호출자 책임
· 필수:
Modbus로 제어하기 위해서는 PANATERM의 매개변수 설정에서 Pr6.28 특수 기능 상태를 1 (Modbus 통신 기동에 의한 Block 동작 유효)로 변경해야 한다. 원점복귀 없이 절대 위치 제어를 사용하려면 Pr60.55=1(강제 원점 확정)로 바이패스가 필요하나, 현재 Err94.2가 재현되고 있어 하드웨어 디버깅이 진행 중이다.
